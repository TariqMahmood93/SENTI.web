About

What is SENTI?

SENTI (SENTI-based Data Imputation) is a data imputation system that uses sentence transformer embeddings to semantically recover missing values in incomplete relational tables — without any model training or labeled data.

Imputation Pipeline
1
Incomplete Dataset R₀
Input with missing values (⊥)
2
Sentence Embedding
Encode tuples via SENTI / LaBSE
3
Semantic Similarity Search
FAISS k-NN over embedded space
4
Value Propagation
Majority vote / weighted imputation
5
Complete Dataset R₀*
Imputed output ready for downstream use
🎯
Zero-Shot Imputation
No pre-training or labeled missing data required. SENTI leverages pre-trained sentence transformers to impute values purely from semantic context.
📐
Relational Semantics
Treats tuples as natural language sentences, capturing inter-attribute relationships that tabular models miss.
⚡
Incremental Support
Handles streaming data with online index updates — new tuples are imputed without reprocessing the entire dataset.
🔬
Evaluation Suite
Built-in metrics (accuracy, F1, RMSE) compare imputed vs. ground truth, with support for multiple null injection patterns (MCAR, MAR, MNAR).
Methodology

How It Works

SENTI follows a three-phase process — from raw incomplete data through semantic embedding to fully imputed output.

01
🗂️
Null Injection & Preprocessing
Missing values are identified or synthetically injected under MCAR, MAR, or MNAR patterns. Columns are selected and tuples are serialized into natural language strings.
02
🧠
Semantic Encoding via SENTI
Each tuple is encoded using a sentence transformer (SENTI or LaBSE). The resulting embeddings capture semantic proximity between records, including those with missing attributes.
03
🔍
k-NN Search & Value Recovery
A FAISS index enables fast approximate nearest-neighbour retrieval. The k most similar complete tuples vote to determine the imputed value for each missing cell.

Architecture

Architecture

SENTI is built as a modular Streamlit application with a clear separation between the UI layer, imputation engine, and evaluation pipeline.

The architecture separates the UI layer (Streamlit frontend) from the imputation engine (sentence transformer + FAISS index) and the evaluation pipeline (metrics computation). In static mode the full index is built once; in incremental mode the index is updated online as new batches arrive — keeping imputation time constant regardless of dataset growth.

Features

Key Capabilities

SENTI combines state-of-the-art NLP with practical data engineering to deliver a complete imputation workflow.

🎯
Zero-Shot Operation
No fine-tuning or labeled imputation examples required. Works out-of-the-box on any relational dataset with pre-trained transformer models.
⚡
Incremental Mode
Efficient online imputation for streaming data. The FAISS index is updated incrementally, enabling constant-time imputation for new tuples.
🔬
Multiple Null Patterns
Supports MCAR, MAR, and MNAR missing data generation with configurable rates, target columns, and conditions for realistic evaluation.
📊
Built-in Evaluation
Comprehensive metrics including accuracy, F1-score, precision, recall, and RMSE for categorical and numerical columns respectively.
🌐
Multilingual Support
LaBSE sentence transformer enables imputation across multilingual datasets, handling heterogeneous text in multiple languages seamlessly.
🖥️
Interactive UI
Full Streamlit web application with visual diff highlighting of imputed cells, dataset statistics, and downloadable results in multiple formats.
